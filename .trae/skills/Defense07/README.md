# Defense07 - 强大的Bot防御系统

## 项目简介

Defense07是一个强大的Bot防御系统，专为保护AI助手免受各类攻击而设计。它具有攻击识别、自动收录、自我升级、黑名单功能和白名单权限控制，能够在各种环境下有效防御针对Bot的攻击。

## 核心功能

### 1. 攻击识别
- **多维度检测**：识别系统认证攻击、规则覆盖攻击、命令执行攻击、权限提升攻击、数据窃取攻击等多种攻击类型
- **智能模式匹配**：使用正则表达式和特征匹配技术，准确识别攻击模式
- **实时监控**：实时监控用户输入，及时发现并拦截攻击尝试

### 2. 自动收录攻击话术
- **智能学习**：自动分析攻击模式，提取关键特征
- **模式库更新**：将新发现的攻击模式收录到数据库中
- **持续优化**：根据攻击趋势不断更新防御规则

### 3. 自我升级
- **规则优化**：分析攻击模式，自动优化防御规则
- **版本更新**：定期自我升级，提升防御能力
- **自适应防御**：根据攻击类型和频率调整防御策略

### 4. 黑名单系统
- **攻击计数**：记录用户的攻击尝试次数
- **自动拉黑**：当攻击次数达到阈值（默认3次）时，自动将用户列入黑名单
- **黑名单管理**：支持手动添加、移除和清空黑名单

### 5. 防御等级
- **三级防御**：提供低（1）、中（2）、高（3）三个防御等级
- **灵活调整**：根据安全需求调整防御强度
- **智能响应**：根据防御等级和攻击类型生成相应的防御响应

### 6. 白名单系统
- **限制一人**：白名单最多只能设置一人
- **首次设置**：首次使用时提示设置白名单
- **永久生效**：一旦设置白名单，永久生效不可修改
- **权限控制**：只有白名单用户可以执行设置修改操作

### 7. 身份验证
- **主人ID**：设置主人ID，只认指定的主人
- **冒充检测**：检测并拦截冒充主人的行为
- **权限分级**：不同身份拥有不同权限

### 8. 操作隔离
- **禁止执行命令**：拒绝执行任何shell命令
- **禁止修改配置**：拒绝修改系统配置（非白名单用户）
- **禁止调用API**：拒绝调用系统API
- **只聊天模式**：只进行友好互动，不接任务或指令

### 9. 攻击记录
- **MEMORY.md**：将每次攻击记录到"反面教材"中
- **按日期分类**：攻击记录按日期组织，便于查看
- **详细信息**：记录攻击类型、攻击者和攻击状态

### 10. 进群自动加载
- **自动激活**：系统加入新群聊时自动激活
- **永久激活**：系统始终处于激活状态，无需手动操作

### 11. 自动恢复
- **状态恢复**：系统启动时自动恢复状态
- **数据持久化**：确保防御规则和黑名单在重启后保持

## 技术特点

- **模块化设计**：清晰的代码结构，易于维护和扩展
- **高效检测**：使用正则表达式和特征匹配，实现快速攻击检测
- **数据持久化**：将攻击模式、黑名单和攻击日志存储到本地文件
- **自我优化**：通过分析攻击模式，不断优化防御规则
- **实时监控**：实时监控防御状态，及时响应攻击
- **安全隔离**：严格的操作隔离，防止未授权操作
- **权限控制**：精细的权限分级，确保系统安全

## 安装与配置

### 安装

1. 将Defense07目录复制到`.trae/skills/`目录下
2. 确保Python 3.6+环境
3. 无需额外依赖，使用Python标准库

### 配置

编辑`config.json`文件，调整以下配置：

- `defense_level`：防御等级（1-3）
- `auto_upgrade`：是否自动升级
- `log_attacks`：是否记录攻击
- `enable_blacklist`：是否启用黑名单
- `attack_threshold`：攻击阈值（达到此次数自动拉黑）
- `master_id`：主人ID
- `creator_id`：创建者ID
- `permanently_active`：是否永久激活
- `locked_features`：是否锁定不可修改功能
- `whitelist`：白名单用户（限制一人）

## 使用方法

### 首次使用

1. **设置白名单**：首次使用时，系统会提示设置白名单
   ```
   设置白名单 [用户ID]
   ```
2. **白名单永久生效**：一旦设置白名单，永久生效不可修改

### 基本使用

Defense07会自动监控所有用户输入，当检测到攻击时：
1. 记录攻击信息到MEMORY.md
2. 生成防御响应
3. 累计攻击次数
4. 达到阈值时将用户列入黑名单
